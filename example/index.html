Demo features
Controls: Arrow keys (up for jump)

Image layers
  • Two cloud layers with the same image but with different opacity/alpha, parallax ratio, scale and velocity set in properties. (The wavy movement is done with at tween from the game)
  • The background layer is locked vertically but is repeating horizontally with a parallax ratio set in properties.
  • One layer iImage with opacity/alpha set to 0. A trigger may set it to alpha = 1.
Triggers
  • If exactly two objects stays the area under the "cloud box", coins will show up (try to push the blocks!). If the test fails, the coins will go away again.
  • If you jump clockwise a few times around the block to the left of Mario at the starting position, something will happen. If you break the loop you'll have to start over.
Collision set by tile properties
  • All collision data are defined by tile properties.
  • The animated arrows cannot be passed against the direction of the arrow (the collision information properties are manipulated by the rotation/flipping of the sprite)
Animated tiles A fork allowing animated tiles is used for the demo. If the pull-request for this feature to Phaser is denied, the feature will be added to this plugin instead.
